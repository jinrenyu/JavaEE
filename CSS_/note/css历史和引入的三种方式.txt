1. 什么是CSS.
CSS的英文全称叫Cascading Style Sheet, 中文翻译过来也叫'层叠级联样式表'.
它是于1996年发布定制初稿,然后在1998年的时候发布了CSS 2.0的版本, 接着又在2004年
发布了CSS2.01版本, 这个版本在2.0基础上最了一些修正, 是我们j2ee人员使用最多的标准.
2010年的时候发布了CSS 3.0版本, 3.0版本主要添加了一些动画、图像纠正、作图、3D等功能,
3.0版本的内容主要用于专门的前端开发,它可以使实现一些非常炫酷的网页内容.我们课堂上简单介绍一些.
因为3.0版本的特性, 各个浏览器支持的情况不一样(特别是IE)对其支持不太友好, 所以当使用这些
属性的时候开发人员需要了解浏览器的一些针对其功能的特性.

2. CSS的作用
它可以帮助我们实现"逻辑数据"和"表现数据"的分离. HTML 5也建议我们只使用标签来构建
网页的框架数据, 而又专门的CSS来控制其表现的形式. 它可以帮助我们更好的管理网页的风格.

2.5 CSS文档中的注释表示方法以及基本语法
它使用 /*注释内容*/ 的方式来表示一行注释
Eclipse中可以使用Ctrl+Shift+/的方式来快速添加注释

它的基本语法格式为:
selector(选择器) {
    css的属性: css属性的值;
}

末尾的分好可以不加, 但是经验老道的老司机的都会添加上最后的分号,
好处是可以避免下次再往最后一行后面添加新的属性的时候忘记添加分号,
多个属性之间使用分号(;)进行分割

例如:
p {
    /* 改变字体的颜色为红色 */
    color: red;
}

3. CSS加载的分类
1). 外联式(外部样式)

a. 第一步, 在head标签之中添加link标签
<link rel="stylesheet" type="text/css" href="链接css的url" />

b. 第二步, 书写对应的css外部文件

优点: 它是最契合HTML 5建议的网页中"表现形式"和"逻辑数据"进行分离的一种引入形式,
书写的网页代码最为美观也最容易维护和控制, 一般大型网站又比较穷的公司都使用该种方式
缺点: 需要引入外部外部文件, 当文件过多时会影响网页的加载速度

2.) 内嵌式(内部样式)

使用内部样式只需要在head标签中添加style标签,在style标签中书写样式属性和其对应的值:
<style type="text/css">
    ......
</style>

优点: 不需要引入外部文件, 所有的样式代码都书写在当前网页文档中, 速度快, 效率高
缺点: 维护起来相对于外部方式显得比较繁琐, 需要有专门人员对这种形式的样式进行更新和维护,
一般大型的门户网站公司会采用该种方式, 因为比较起人力成本, 这些网站的加载性能会更为重要一些,
说白了就是有钱任性,他们不在意这些人力成本.

3.) 内联式(行内样式)
优点: 用在非常特殊的标签表现形式或者标签的样式不会或者很少发生改变的时候, 它具有最高的加载优先级
缺点: 维护起来非常非常麻烦, 尽量少的使用.

4.) (补充)引入式
使用方式: 
在css代码中(不是某个选择器内)使用专门的css命令:
@import url("url路径");
其中url可以省略.
必须写在<style type="text/css">之中

优点: 可以用于组织和管理非常大量的css文件
缺点: 因为其效果和外部样式相同并且早期(特别是IE)在使用该种方式的时候会出现闪屏
所以这种方式使用的比较少

4. 样式加载的优先级别
行内样式 > 内部样式 ~ 外部样式  ~ 引入式(导入样式)
注意: 内部样式的优先级高于外部样式的前提是, 内部样式在网页中出现的位置要在
外部样式之后(类似于数据库中'后赢'的机制), 因为文档的加载顺序是由上至下的
